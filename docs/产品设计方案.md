## 一、产品定位

### 1.1 核心价值主张

```
「不只是聊天，能真正帮你干活」
```

| 维度 | 说明 |
|------|------|
| **目标用户** | 25-40 岁中国大陆办公白领 |
| **核心场景** | 办公提效、数据处理、文案写作、信息搜集 |
| **差异化** | 能执行复杂任务，不是单纯的聊天机器人 |
| **使用门槛** | 零门槛，一键安装，开箱即用 |

### 1.2 用户画像

#### 典型用户：小李

| 属性 | 描述 |
|------|------|
| **年龄** | 28 岁 |
| **职业** | 互联网公司运营专员 |
| **城市** | 深圳 |
| **日常工作** | 数据分析、周报撰写、活动文案、跨部门沟通 |
| **痛点** | 重复性工作多、加班写报告、数据整理耗时 |
| **付费意愿** | 愿意为提效工具付费，预算 20-50 元/月 |
| **手机** | iPhone 14 / 华为 Mate 50 |
| **使用习惯** | 微信重度用户、喜欢简洁界面、讨厌复杂设置 |

### 4.2 技术栈选型

| 模块 | 技术选型 | 理由 |
|------|----------|------|
| **前端框架** | uniapp | 一套代码多端发布，国内生态好 |
| **UI 组件** | uView UI | 成熟、文档全、中文支持 |
| **状态管理** | Pinia | 轻量、易用 |
| **云侧服务** | OpenClaw Gateway | 复用现有代码 |
| **大模型** | 通义千问 + 智谱 GLM | 国内访问快、成本低 |
| **搜索 API** | Tavily + 百度 | 多源搜索结果更全 |
| **数据库** | PostgreSQL + Redis | 成熟稳定 |
| **文件存储** | 阿里云 OSS | 国内访问快 |
| **消息推送** | 极光推送 | 国内稳定 |
| **内容审核** | 阿里云内容安全 | 合规要求 |

### 4.3 核心 API 设计

#### 对话接口

```javascript
// WebSocket 实时对话
POST /api/v1/chat
{
  "user_id": "user_123",
  "message": "帮我分析这个 Excel",
  "file_id": "file_456",
  "session_id": "session_789"
}

// 响应 (流式)
{
  "type": "chunk",
  "content": "正在分析...",
  "status": "processing"
}
```

#### 文件上传接口

```javascript
POST /api/v1/files/upload
Content-Type: multipart/form-data

{
  "file": <binary>,
  "user_id": "user_123",
  "type": "excel"
}

// 响应
{
```
